# Support

## Getting Help 🛟

- Read `README.md` first for setup and runtime basics.
- Use `docs/operations-runbook.md` for incident triage.
- Use `docs/extensions.md` for feature-extension patterns.

## Channels

- **Bug reports & feature requests:** [GitHub Issues](https://github.com/johndotpub/clawsprawl/issues)
- **Usage questions:** Open a GitHub Issue with the `question` label.
- **Security vulnerabilities:** See `SECURITY.md`

## Bug Reports

When filing an issue, include:

- What you expected
- What happened
- Logs/error snippets (redacted)
- Exact commands run (`npm run test`, `npm run build`, etc.)

## Feature Requests

Please include:

- User story and problem statement
- Proposed approach
- Risks and compatibility concerns

## Security Reports

For sensitive issues, follow `SECURITY.md` and avoid public disclosure first.
